# AI Tagger for Obsidian

AI Tagger generates tags for notes using a configurable AI endpoint. It can call a local Ollama server or an OpenAI-compatible chat completions endpoint, favoring tags that already exist in your vault.

## Features

- Generate tags for the active note from the command palette.
- Generate tags from the ribbon button or a note's file menu.
- Optionally regenerate tags automatically after markdown files change.
- Choose tag casing: keep AI output, lower kebab, UC first, camel case, Pascal case, or snake case.
- Preserve manual tags while replacing tags that were previously generated by the plugin.
- Prefer existing vault tags when they match the AI's suggestion, even if the selected casing format differs.

## Development

Install dependencies and build:

```bash
npm install
npm run build
```

For watch mode:

```bash
npm run dev
```

To load in Obsidian, place this directory at:

```text
<vault>/.obsidian/plugins/obsidian-ai-tagger
```

Then enable **AI Tagger** under **Settings -> Community plugins**.

## Sharing test builds

Every push to `main` runs the GitHub Actions build and uploads `obsidian-ai-tagger.zip` as a workflow artifact. For a more permanent test build, push a tag such as `v0.1.0`; the workflow will create or update a GitHub release with the zip plus the individual Obsidian plugin files.
